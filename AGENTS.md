## Brief overview

These are general guidelines for working in this workspace.

## Communication style

- Be direct and concise in your responses.
- Provide clear explanations for your actions.

## Development workflow

- Use the `docs/` directory for all documentation.
- Follow the existing structure of the project when adding new files.
- Use the `justfile` for commands 

## Coding best practices

- Use PEP-8 following all ruff defaults
- Write tests for new features.

## Project context

- This project is a Python-based CLI tool for uploading images to Google Photo albums
- The project is hosted on GitHub at https://github.com/pydanny/google-photos-uploader

## Development Environment

- The development environment is Mac-based.
- uv is the tool for Python environment and dependency management
- ruff is used to lint and format the code
